Migracja startupu MedTech do Google Cloud Platform

Platforma
Google Cloud Platform

Wyzwanie

Startup rozwijający platformę do zdalnego monitorowania pacjentów (RPM) działał na współdzielonym hostingu, który nie spełniał wymagań regulacyjnych (RODO, MDR) ani wydajnościowych. Aplikacja mobilna obsługiwała 12 000 aktywnych pacjentów, a dane medyczne wymagały szyfrowania i audytowalnego dostępu.

Klient potrzebował infrastruktury, która skaluje się automatycznie podczas porannych szczytów (pacjenci wysyłają dane o 7:00-9:00), zapewnia 99.95% SLA i spełnia wymogi certyfikacji ISO 13485.

Rozwiązanie

Zaprojektowaliśmy architekturę cloud-native na GCP z pełnym wykorzystaniem usług zarządzanych:

  1. Konteneryzacja na GKE — backend (Python/FastAPI) i mikroserwisy przeniesione do Google Kubernetes Engine z autoscalingiem opartym na metrykach custom (liczba aktywnych sesji pacjentów).
  2. Cloud SQL z Private IP — baza PostgreSQL z automatycznym backupem, replikacją i dostępem tylko przez prywatną sieć VPC.
  3. Cloud Run dla API mobilnego — serverless endpoint obsługujący requesty z aplikacji mobilnej, skalujący się od 0 do 500 instancji w sekundy.
  4. Security & Compliance — VPC Service Controls, Cloud KMS do szyfrowania danych pacjentów, Cloud Audit Logs do pełnej audytowalności.

CI/CD i DevOps

Wdrożyliśmy pipeline CI/CD oparty na Cloud Build z automatycznym budowaniem obrazów Docker, skanowaniem podatności (Artifact Registry vulnerability scanning) i deploymentem do GKE przez GitOps (Config Sync). Każdy commit przechodzi testy jednostkowe, integracyjne i security scan przed wdrożeniem.
