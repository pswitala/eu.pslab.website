Wyzwanie

Uczelnia wyższa z 5000+ studentów i 400+ pracowników akademickich borykała się z przestarzałą infrastrukturą IT. Kluczowe problemy:

  • Rozproszone systemy tożsamości — oddzielne konta w Active Directory, systemie dziekanatowym, platformie e-learningowej (Moodle) i bibliotece. Studenci zarządzali 4-5 różnymi loginami.
  • Starzejące się serwery — trzy serwerownie z serwerami 8-12 lat, brak redundancji, ryzyko utraty danych.
  • Brak cyberbezpieczeństwa — brak SIEM, brak centralnego monitoringu, incydenty phishingowe co tydzień.
  • Wymagania obliczeniowe — laboratoria naukowe potrzebowały klastra HPC (High-Performance Computing) do obliczeń, który nie mógł być całkowicie w chmurze ze względu na wymagania dotyczące latencji i kosztów GPU.

Rozwiązanie — architektura hybrydowa

Zaprojektowaliśmy hybrydowe środowisko IT łączące zalety chmury (Azure) z niezbędną infrastrukturą on-premise:

Chmura (Azure)

  • Azure AD + Azure AD Connect — synchronizacja on-premise AD z Azure AD. SSO do wszystkich systemów (Moodle, USOS, biblioteka, poczta).
  • Microsoft 365 A3 for Education — poczta Exchange Online, Teams, OneDrive, SharePoint dla 5400+ użytkowników (studenci + pracownicy).
  • Azure App Proxy — bezpieczny dostęp do aplikacji on-premise (USOS, system biblioteczny) bez VPN.
  • Azure Virtual Desktop (AVD) — laboratoria zdalne ze specjalistycznym oprogramowaniem (MATLAB, AutoCAD, SPSS).
  • Microsoft Sentinel — cloud-native SIEM do centralnego monitoringu bezpieczeństwa całej infrastruktury.

On-premise

  • Klaster HPC — 12 serwerów z GPU NVIDIA A100 do obliczeń naukowych. Bezpośrednie podłączenie do instrumentów laboratoryjnych.
  • Sieć kampusowa — redesign sieci: Cisco Catalyst core switches, FortiGate firewall, Ubiquiti UniFi WiFi 6 (120+ access pointów).
  • Active Directory — utrzymany on-premise z hybrid join do Azure AD.
  • Backup — strategia 3-2-1: lokalne backup + Azure Backup (off-site).

Bezpieczeństwo i NIS2

Uczelnia jako instytucja z sektora edukacji podlega dyrektywie NIS2. Wdrożyliśmy:

  • Microsoft Sentinel — centralny SIEM agregujący logi z Azure, firewalli, serwerów on-premise i urządzeń sieciowych.
  • Threat intelligence — automatyczne korelowanie alertów z bazami zagrożeń.
  • Incident response playbooki — zautomatyzowane reakcje na typowe incydenty (blokowanie IP, izolowanie urządzenia).
  • MFA everywhere — Conditional Access wymuszający MFA dla wszystkich użytkowników.

Podejście hybrydowe pozwoliło: zachować klaster HPC on-premise (niskie latencje, kontrola kosztów GPU) przy jednoczesnym przeniesieniu produktywności, tożsamości i bezpieczeństwa do chmury. Uczelnia oszczędza ~35% kosztów IT w porównaniu do full on-premise.
